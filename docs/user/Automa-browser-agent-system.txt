📘 The Easy Ecom (EE) Internal Agent BlueprintThis document explains how to turn your laptop into an automated operations hub that talks to Flipkart and your Supabase database.1. The Core Architecture: "Visual Block" LogicAutoma doesn't use code; it uses Blocks. Every workflow starts with a Trigger and follows a path of Actions.The Trigger: Can be manual (a button you click in EE), scheduled (every morning at 9:00 AM), or a specific event (whenever you open Flipkart).The Action: Clicking a button, typing a date, or downloading a file.The Logic: "If/Else" blocks. Example: If "Stock < 10", then "Email Gurgaon Warehouse", else "Continue".2. Handling Complex Tasks (Filters & Navigation)You aren't limited to simple clicks. You can "Stitch" complex operations together:Multi-Step Filtering: * Action: Drag a "Forms" block to type into the search bar.Action: Use a "Select" block to choose "RTO" from a dropdown.Advanced: You can tell the agent to "Loop." It can take a list of 50 Order IDs from your Gurgaon warehouse and check each one on Flipkart automatically.Cross-System Navigation:Automa can open Tab 1 (Flipkart), copy the Settlement Amount, then open Tab 2 (Your Supabase App) and paste it into the correct field.3. Talking to Supabase (The "EE" Connection)This is how you make your internal tool "Pro." You use the HTTP Request Block.Field in Automa BlockWhat to put thereMethodPOST (to send data) or GET (to read data)URLYour Supabase Project URL (e.g., https://xyz.supabase.co/rest/v1/orders)HeadersYour apikey and Authorization (Service Role Key)BodyThe data you scraped (e.g., {"order_id": "{{orderID}}", "status": "shipped"})Why this works: The agent scrapes the data from Flipkart and "pushes" it into your Supabase database instantly. Your Easy Ecom dashboard then refreshes and shows the live data.4. Avoiding the "Flipkart Block" (Anti-Detection)To keep your Gurgaon IP and Flipkart account safe, follow these "Human-Mimic" rules:Random Delays: Never click buttons at the exact same speed. Use a "Delay" block with a random range (e.g., 3 to 7 seconds).Visual Interaction: Use the "Cursor" block. It makes the browser actually "scroll" and "move the mouse" to the button before clicking, which looks human to Flipkart’s security.Active Session: Since it's a browser extension, you log in once manually. The agent never sees your password; it just uses your active, "safe" session.5. The "Laptop Warehouse" SetupSince you are running this from a laptop in Gurgaon, follow these hardware settings:Power Settings: Set "Lid Close Action" to "Do Nothing". Plug it into a permanent power source.The "Awake" Rule: Use a tool like PowerToys Awake to ensure the laptop never goes to sleep.Background Running: Automa can run in a "Hidden" window. You can keep working on your consulting or Easy D2C in one window while the "Agent" processes Flipkart orders in another.How to Build Your First "Easy Ecom" WorkflowRecord: Hit the Record button and download a Settlement Report.Clean: Remove any "extra" clicks the recorder caught.Logic: Add an HTTP Request Block at the end to send that file's data to Supabase.Schedule: Set the Trigger Block to "Daily at 10:00 AM."
